# 信息安全管理制度（模板）

> 复制到贵司制度体系，由信息安全负责人审批后生效。 `[方括号]` 处请替换。

## 1. 目的

规范 `[公司名称]` 统一身份认证平台（ZestSSO）的建设、运维与使用，满足业务安全与合规要求。

## 2. 适用范围

适用于所有使用 ZestSSO 的业务系统、管理员、运维人员及第三方集成开发商。

## 3. 组织与职责

| 角色 | 职责 |
|------|------|
| 信息安全负责人 | 制度审批、等保对接、事件定级 |
| IAM 管理员 | 用户/客户端/身份源管理，权限复核 |
| 运维工程师 | 部署、备份、监控、补丁 |
| 研发负责人 | RP 接入评审、Secret 管理 |

## 4. 账号管理

1. 禁止共用 Admin 账号； MFA 必须启用。
2. 入职开户通过 SCIM 或 HR 审批流；离职 24 小时内禁用账号。
3. 每季度执行 Access Review（ZestSSO Admin → 权限复核活动）。
4. OAuth 客户端 `client_secret` 每 `[12]` 个月轮换。

## 5. 认证与访问控制

1. 对外系统必须使用 HTTPS + OIDC/OAuth2 接入。
2. 禁止在代码库、聊天记录中明文存储密钥。
3. 高权限 API（`/api/admin/**`）仅允许办公网 / VPN 访问（网络层策略）。

## 6. 审计与日志

1. 审计日志保留不少于 `[180]` 天。
2. 每月导出审计抽样检查异常登录、批量改密。
3. 日志不可被普通管理员删除。

## 7. 备份与应急

1. MySQL 每日备份（`scripts/backup-prod.*`），保留 `[30]` 天。
2. 每半年演练恢复一次并记录。
3. SSO 不可用超过 `[30]` 分钟启动应急预案（业务只读 / 本地降级登录策略由业务方定义）。

## 8. 变更管理

1. 生产变更需工单；禁止周五晚重大升级。
2. 升级前执行 `scripts/acceptance.ps1`。
3. Flyway 迁移须在预生产验证。

## 9. 第三方与外包

集成开发商须签署保密协议；仅授予最小权限测试客户端。

## 10. 违规处理

按公司人事与信息安全奖惩规定执行。
