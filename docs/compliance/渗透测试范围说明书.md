# ZestSSO 渗透测试范围说明书

> 委托第三方安全厂商进行 Web 应用渗透时使用。可根据合同附件直接引用。

## 1. 测试目标

| 项 | 说明 |
|----|------|
| 系统名称 | ZestSSO 统一身份认证平台 |
| 版本 | 1.0.0-SNAPSHOT（以实际部署 tag 为准） |
| 环境 | **预生产** 或 **生产**（需甲方书面授权） |
| 入口 URL | `https://<sso-domain>/` |

## 2. 测试范围（In Scope）

### 2.1 Web 应用

- 登录页 `/login`、MFA `/login/mfa`
- Admin 控制台 `/admin/**`
- Admin API `/api/admin/**`
- 公开 API `/api/public/**`
- OAuth2/OIDC 端点 `/oauth2/**`、`/userinfo`、`/connect/logout`
- SCIM `/scim/v2/**`（若已暴露）
- Passkey 端点 `/api/public/webauthn/**`

### 2.2 认证与协议

- OAuth2 授权码 + PKCE 流程
- Client Credentials（若开放）
- Device Authorization Grant（`device-cli`）
- Back-Channel / Front-Channel Logout
- 联邦登录（已配置的外部 IdP 按钮）

### 2.3 基础设施（只读验证）

- TLS 配置（版本、套件、HSTS）
- HTTP 安全头（CSP、X-Frame-Options 等）
- 错误信息泄露

## 3. 排除范围（Out of Scope）

- 对 MySQL / Redis 端口的直接暴力破解（除非甲方明确要求内网渗透）
- 社会工程学、钓鱼
- DDoS 压力测试（单独性能测试项）
- 下游业务系统（ZestFlow、ZestLLM 等 RP，除非另签合同）

## 4. 测试方法

- OWASP WSTG / ASVS Level 2 为基准
- OWASP Top 10（2021）全覆盖
- OAuth/OIDC 专项：redirect_uri 篡改、PKCE 绕过、token 泄露、logout CSRF

## 5. 交付物

- [ ] 渗透测试报告（含风险等级、复现步骤、修复建议）
- [ ] 漏洞清单 Excel
- [ ] 复测报告（整改后）

## 6. 风险定级标准

| 等级 | 说明 | 整改时限建议 |
|------|------|--------------|
| 严重 | 未授权接管 Admin、任意用户登录 | 7 天 |
| 高 | OAuth redirect 绕过、SCIM 未授权写 | 14 天 |
| 中 | 信息泄露、弱配置 | 30 天 |
| 低 | 最佳实践 | 下一版本 |

## 7. 甲方配合

- 测试账号：普通用户、TENANT_ADMIN、SSO_ADMIN 各一（测评后回收）
- OAuth 测试客户端 `pentest-client` 及 redirect `https://pentest.example/callback`
- 测试时间窗口与应急联系人
- 授权书（盖章）
